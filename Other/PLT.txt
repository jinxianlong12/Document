产品技术
      1. 前端使用国际标准WEB Standard HTML5(Canvas, WEBGL, WEB Component)Java, JavaScript开发, 无需安装额外插件. 
      2. 前端脚本语言使用谷歌主推的框架技术polymer, 产品使用最新框架polymer3, 稳定的结构, 丰富的页面效果, 提高用户体验.
      3. 服务端用JAVA开发, 提高扩展性, 兼容性, 系统管理人员无需查看复杂的文档，只需在界面上管理系统.

易用性
      1. 产品门槛低, 可便快捷地设计, 拖拽的形式设计界面, 无需开发.
      2. 简单绑定多种复杂数据, 快速开发, 实时的数据显示.
      3. 提供丰富的2D,3D的图形工具.
      4. 强大的Chart组件, 结合仪表, 表格等BI组件, 实现复杂的报表.
      5. 定制化的3D仓库组件, 快速实现3D仓库.
      6. 提供丰富的表单组件, 快速定制多种条件搜索.
      7. 模拟生产, 仓库, 物流等现场, 结合动画效果及数据, 提高用户体验.

数据交互
      1. 产品支持WebService, MQTT, STOMP, firebase等多种多种数据交互方式, 实时动态的显示数据.
      2. 简单配置多个数据库对接, 支持大部分关系型数据库(Oracle, MSSQL, MySql, PostgreSQL, MonggoDB, Cassandra..等等), 一个系统显示多个系统的数据.
      3. 设定数据发送/接收时间, 不同业务模式设置不同的时间, 提高产品性能, 提高实时性. 
      4. 无需通过数据库存储, 实时接收分析IoT感应器的数据, 分析数据, 可视化显示.
      5. 支持读取Excel格式的数据源, 无需导入文件, 拖拽Excel文件到产品界面即可读取数据.
      6. 双向的数据交互, 接收数据, 也可发送数据.

软件特点
      1. 产品是B/S(浏览器/服务器)架构, 跨平台, 无论是Window,Linux, Mac系统, 用户无需安装程序, 浏览器输入产品地址访问产品.
      2. 移动设备无需安装相应的Android, IOS等额外License, 移动端浏览器访问产品.
      3. 产品支持丰富的动画效果.  
      4. 提供丰富的播放功能, 分屏播放, 全屏播放, 定时滚动.
      5. 产品是B/S模式, 可部署到云上.
      6. 支持看板的权限管理, 版本管理.

部署

Label:  X-Label是标签设计及管理工具, B/S模式的系统, 国际标准web语言来开发的, 支持ZPL打印指令. 目前很多设计标签工具C/S结构的系统, 特别是用flex开发的比较多,
        flex比起web沉重, 而且官方已经停止支持了, 维护难, 升级不支持. 如果购买之后支持的标签种类不多, 不够满足业务的需求. 这时我们要添加标签种类的话就会很麻烦了
        或者用zpl 代码来设计标签, ZPL代码是标签原始语言, 用这个设计的话 专业性要求高, 维护困难, 设计不方便, 管理业不方便.
        还有一个难题就是标签系统的重复投资,
        一般一个工厂会有不同的系统会用到标签的, 比如erp, mes, wms等等  每个系统都配一个标签设计软件的话是投资的重复, 费用高, 而且管理也分开,
        X-label设计是拖拽组件的方式来设计标签, 而且简单的实现和多种系统的对接, 他是用一个系统来管理所有标签的
        X-Label 是b/s结构, 是跨平台的, 无需安装, 用浏览器输入地址就可以连接到设计界面.

Mobile: X-Mobile是手机版的MES, 对于MES起辅助作用, 那些常用的主数据, 报表等用移动设备来显示,
        还有那些现场来回, 执行多个工序操作的时候, 来回移动多的时候用X-mobile来解决这样的问题
        也是X-MES的扩展与补充, 集合移动设备的特性实现电脑端无法实现的功能, 像那些NFC, 扫描, 消息推送, 拍照上传等, MES无法实现的, 用Mobile可以实现.
        像设备维修, 产品不良的时候直接拍照上传到系统, 工序设备上贴上nfc, 无需输入选择, 只需靠近手机来自动获取数据
        扫描Lot, 直接读取Lot的进度或执行Lot操作
        支持信息推送.

        X-mobile是用Hybrid APP模式, 一般手机应用较多的是native app或web app.native APP是... web app是...
        Hybrid APP具备“Native App良好用户交互体验的优势”和“Web App跨平台开发的优势”.
        它是安装类型的, 但通过Cordoba, 安卓和IOS系统通信, 不需要像安卓用安卓语言, IOS用Iphone语言那样,
        无需开发两个客户端, 只需维护一个客户端, 从而节省开发成本, 维护成本.
        客户端是用国际标准web语言俩开发, 和已有的Mes plus系统的服务端进行通信, 所以服务端开发是很少的, 除了特别需求的界面外大部分是使用以开发的服务.
        需要开发的前端界面, 高度模块化，组建重用度高.
        因为X-mobile是MES, WMS等业务的辅助和扩展应用, 开发起来会很快的.
        支持国际化, 目前是中韩英, 根据需要, 在配置界面添加另外的语言.

        界面说明分三个部分, 主数据, 操作, 报表三个方面来说.

        仓库方面使用的比较多, 各种扫描, 可以代替PDA, 比PDA功能多, 像NFC, 图片上传, 信息推送 等等等, 用一个移动设备来完成多种设备实现的功能.


        手机版的MES, 或是WMS 把系统和移动设备的特性结合起来
        移动设备有哪些特性呢? 硬件方面是设备小, 可随意移动,
        软件方面的特点-->扫描, 扫描用的地方很广, SMT产线操作时  扫描批次, 扫描托盘, 扫描设备等等来获取信息
                            仓库业务的话会有更多的地方用到, 入库, 上架下架, 盘点, 包装, 出库等等等都需要用到
                      拍照 不良信息直接上传到系统
                      NFC  近距离感应技术, 标签贴到设备或某个地方, 把移动设备贴到标签上就获取标签里面的信息
                      信息推送, 用移动设备实现信息推送, 实时的处理审核审批等业务
                      还有下载对讲应用的话现场里实现对话的功能
                      报表, X-Monitoring设计的报表也可以用移动设备浏览
                      PDA只负责扫描, 画面简单, 不可能实现以上的功能, X-Mobile可以把PDA业务完全覆盖掉, PDA和X-Mobile比的话可以是以前打电话手机和智能手机的区别

        结构是Hybrid应用, 介于普通应用和网页应用之间, 普通安装的应用的用户体验好, 速度快, 网页应用跨平台
        系统框架, 和开发语言 和X-MES一样 Server是java客户端是JavaScript, 无需学安卓等另外的开发技术
